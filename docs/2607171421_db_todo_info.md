# 🔑 Найважливіша рекомендація — зберігання файлів

Поточний `bytea` у БД спричинить **значне зростання розміру БД та зниження продуктивності**. Рекомендую гібридний підхід:

```sql
storage_type smallint DEFAULT 1,  -- 1=DB, 2=FileSystem, 3=S3
value bytea,                       -- для малих / inline
file_path varchar(500)             -- для зовнішнього сховища
```

Це дозволить спочатку зберігати в БД, а потім мігрувати без зміни схеми.
